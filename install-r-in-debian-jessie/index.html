在Debian上安装R

R Dec 22, 2017 我勒个去

近期打算开始入手时间序列了,而这方面R语言比Python更加方便,无论是书还是库都资源更多一些。
下面我们主要介绍下如何在Debian的jessie上安装R的3.4.3版本。
R语言相比Python1个很不好的地方在于库的依赖完全就是瞎搞,有些库很难找到对应的依赖包。
原本jessie默认的R的版本是3.1.1的,但是在安装TSA库的时候发现有些依赖根本装不上去。于是使用如下的方式进行升级: 

  
deb http://mirrors.tuna.tsinghua.edu.cn/CRAN/bin/linux/debian stretch-cran34/

这里在系统源中添加这么一行,然后执行如下的方式: 

  
apt-get update  
apt-get install r-base r-base-dev

然后就可以安装3.4.3版本的了。但是很遗憾的是,安装TSA库提醒其他库依赖无法解决。
此时有2种方式,1种是手动下载包安装,使用如下的方式: 

  
sudo R CMD INSTALL 对应的包名称

对应包的下载地址可以访问https://cran.r-project.org/src/contrib/。
当然也可以在R中执行: 

install.packages('包的路径')

下面只能手动下载包进行编译了。 

  
wget https://mirrors.tuna.tsinghua.edu.cn/CRAN/src/base/R-3/R-3.4.3.tar.gz  
./configure  
make

需要注意的是,R的3.4.3版本要求curl的版本大于7.22才可以安装成功。另外还依赖Java的JDK和JRE的设置才能成功。
这个编译的过程会持续10来分钟那样,然后再执行上述的安装方式就可以安装成功了。

参考文章:

https://cran.r-project.org/bin/linux/debian/
